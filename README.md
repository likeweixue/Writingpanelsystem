**写作帮手（Writingpanelsystem）**

Writingpanelsystem/
├── 主程序文件
│   ├── main.js          - Electron 主进程
│   ├── preload.js       - 预加载脚本（暴露安全 API）
│   ├── index.html       - 主界面入口
│   ├── package.json     - 项目配置
│   └── jszip.min.js     - ZIP 压缩库
│
├── css/
│   └── style.css        - 主样式表（包含所有主题样式）
│
├── js/                  - 所有 JavaScript 模块
│   ├── app.js           - 应用主入口
│   ├── data.js          - 数据模型（Book/Volume/Chapter）
│   ├── books.js         - 书籍管理核心逻辑
│   ├── ui.js            - UI 渲染和页面切换
│   ├── toolbar.js       - 工具栏功能（导入/导出/排版/校对等）
│   ├── settings.js      - 设置页面（备份/安全/主题）
│   ├── jianghu.js       - 江湖模块（外部链接管理）
│   ├── xuefu.js         - 学府模块（素材库）
│   └── sidebar.js       - 右侧边栏管理
│
├── themes/              - 主题样式
│   ├── default.css      - 默认白
│   ├── eye.css          - 护眼绿
│   ├── warm.css         - 经典黄
│   ├── dark.css         - 暗夜黑
│   └── open.css         - Open 圆润毛玻璃
│
├── html/                - 独立工具面板
│   └── stats.html       - 数据看板
│
└── Help/                - 帮助文档

是一款基于 html，css，js 开发的小说写作软件，以最大程度的适配不同的操作系统，理论上即使是我没有发布 Windows 版本，你依然可以用浏览器打开，内置大纲，数据线，世界观设定，人物设定，笔记，双栏，关系图等小说作者写作常用的功能，助力小说作者早日签约。

开发的核心逻辑是把里面的工具单独开发，形成独立的 html面板，所以理论上接地气的名字是写作面板系统。
**开源协议**
ps；我认为开源是一个项目发展的最好结果，即使是我挂了，项目也不会停滞不前。
关于开源协议，使用的是MIT麻省理工学院协议，通俗易懂就是**你想干嘛就干嘛，出了事别找我。**
1. 你可以干什么（几乎无限制）
随便用：商业软件、个人项目、内部系统都能用。
随便改：可以修改源代码，修复漏洞或增加功能。
随便抄：可以把代码复制到自己的项目里。
随便卖：可以把这份开源代码拿去卖钱，或者把它集成到你自己的收费商业软件里闭源发布（不需要公开你的代码）。

2. 你必须做什么（唯一的硬性要求）
保留版权声明：在你的软件里（比如帮助文档、关于页面或源码注释中），必须保留原我的名字（麻昌生 ）和MIT协议的版权声明。
这相当于：“拿了我的代码，必须提一句这代码是我写的。” 除此之外，没有任何其他强制要求（比如不能使用我的芳名做推广）。

3. 上面都不是主要问题，最核心的“问题”是下面这个，大家都是中国人，彼此之间的德行我还是了解的～
MIT协议明确写了“AS IS”（按原样提供），意思就是：**代码如果出了BUG导致你损失了几个亿，或者造成数据丢失，你不要找我哔哔赖赖的。**
